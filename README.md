# 💳 Financial-Transaction-Risk-Analysis-Credit-Card-Fraud-Detection-project
End-to-end financial fraud detection project using SQL and Python (Pandas, NumPy, Matplotlib, Seaborn) with transaction risk analysis and data visualization.

## 📌 Project Overview
This project focuses on analyzing financial transaction data to identify fraudulent patterns using Data Analytics techniques. The goal is to understand transaction behavior, detect anomalies, and derive business insights that can help in fraud prevention.

---

## 🎯 Objective
- Analyze credit card transaction data
- Identify fraudulent vs normal transactions
- Understand patterns behind fraud cases
- Generate business insights for risk detection

---

## 🛠️ Tools & Technologies Used
- Python
- Jupyter Notebook
- Pandas
- NumPy
- Matplotlib
- Seaborn
- SQL (PostgreSQL / pgAdmin)
- VS Code

---

## 📊 Dataset Information
- Source: Kaggle Credit Card Fraud Dataset
- Features: Time, Amount, Class, V1–V28 (anonymized features)
- Target Variable: Class (0 = Normal, 1 = Fraud)

---

## 📌 Project Workflow

### 1. Data Collection
- Imported dataset from Kaggle

### 2. Data Cleaning
- Checked missing values
- Verified data types
- Ensured dataset consistency

### 3. Exploratory Data Analysis (EDA)
- Analyzed transaction distribution
- Compared fraud vs normal transactions
- Visualized patterns using graphs

### 4. SQL Analysis
- Created database in PostgreSQL (pgAdmin)
- Imported dataset into tables
- Performed basic queries for analysis

### 5. Insights Generation
- Fraud transactions are extremely rare
- Fraud is not dependent only on transaction amount
- Behavioral and time-based patterns are important

---

## 💡 Key Insights
- Fraud cases are highly imbalanced compared to normal transactions
- Transaction amount alone is not a strong indicator of fraud
- Time-based patterns play an important role in detection
